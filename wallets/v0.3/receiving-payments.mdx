## Receiving Methods

| Source | How It Works | Speed |
|--------|--------------|-------|
| **Arkade wallet** | Sender pays your `ark1`/`tark1` address directly | Instant |
| **Lightning** | [Submarine swap](/contracts/lightning-swaps) via Boltz | ~30 seconds |
| **Onchain Bitcoin** | [Boarding flow](/wallets/v0.3/ramps#boarding) (2 confirmations) | ~20 minutes |

## Monitor Incoming Payments

Subscribe to real-time notifications when funds arrive:

```typescript
const stop = wallet.notifyIncomingFunds((notification) => {
  if (notification.type === 'vtxo') {
    for (const vtxo of notification.vtxos) {
      console.log(`Received ${vtxo.amount} sats`)
    }
  }
})

// Later: stop listening
stop()
```

The callback fires whenever new VTXOs arrive at your address.

### Notification Types

| Type | When | Use Case |
|------|------|----------|
| `vtxo` | Offchain payment received | Standard Arkade payments |
| `utxo` | Onchain deposit to boarding address | [Native onboarding](/wallets/v0.3/ramps#boarding) |

### Complete Example

```typescript
import { Wallet, SingleKey, IncomingFunds } from '@arkade-os/sdk'

async function startPaymentMonitor(wallet: Wallet) {
  const stop = wallet.notifyIncomingFunds(async (notification: IncomingFunds) => {
    switch (notification.type) {
      case 'vtxo':
        for (const vtxo of notification.vtxos) {
          console.log(`Received ${vtxo.amount} sats`)
          console.log(`VTXO ID: ${vtxo.txid}:${vtxo.vout}`)
          
          // Update your application state
          await updateBalance()
        }
        break
        
      case 'utxo':
        // Boarding address deposit (needs onboarding to become spendable)
        for (const utxo of notification.utxos) {
          console.log(`Boarding deposit: ${utxo.amount} sats`)
          console.log(`Confirmations needed before onboarding`)
        }
        break
    }
  })
  
  return stop
}

// Usage
const stopMonitor = await startPaymentMonitor(wallet)

// When shutting down
stopMonitor()
```

<Tip>
For web apps, run the payment monitor in a [service worker](/wallets/v0.3/service-worker) so it continues running even when the user closes the tab.
</Tip>

## Address Reuse

Arkade addresses are safe to reuse. Each payment creates a new VTXO, and your address doesn't reveal transaction history to senders.
