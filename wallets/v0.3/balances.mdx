## Quick Balance Check

<CodeGroup>
```typescript TypeScript
const balance = await wallet.getBalance()

console.log('Available:', balance.available, 'sats')  // Spendable now
console.log('Total:', balance.total, 'sats')          // Including recoverable
```

```go Go
balance, err := wallet.GetBalance()
if err != nil {
    log.Fatal(err)
}

fmt.Printf("Available: %d sats\n", balance.Available)
fmt.Printf("Total: %d sats\n", balance.Total)
```
</CodeGroup>

## Balance Fields

| Field | Description | When It Matters |
|-------|-------------|-----------------|
| `available` | Spendable right now | **Primary balance** for payments |
| `total` | Available + recoverable | Full picture of your funds |
| `preconfirmed` | Received but not yet settled onchain | Instant payments waiting for batch |
| `settled` | Anchored to Bitcoin via batch swap | Maximum security state |
| `recoverable` | Expired or sub-dust VTXOs | Needs [recovery action](/wallets/v0.3/vtxo-management) |

<Tip>
For most use cases, display `available` as the user's balance. It represents what they can spend immediately.
</Tip>

### Boarding Balance

If you use [native onboarding](/wallets/v0.3/ramps#boarding), the `boarding` field tracks onchain deposits waiting to become VTXOs:

```typescript
const { boarding } = await wallet.getBalance()

console.log('Unconfirmed deposits:', boarding.unconfirmed)
console.log('Confirmed deposits:', boarding.confirmed)
console.log('Total boarding:', boarding.total)
```

Boarding funds become `available` after onboarding completes (requires 2 confirmations).

## List VTXOs

Inspect individual VTXOs for debugging or advanced coin selection:

<CodeGroup>
```typescript TypeScript
const vtxos = await wallet.getVtxos()

for (const vtxo of vtxos) {
  console.log(`${vtxo.txid}:${vtxo.vout}`)
  console.log(`  Amount: ${vtxo.value} sats`)
  console.log(`  State: ${vtxo.virtualStatus.state}`)
  console.log(`  Batch: ${vtxo.virtualStatus.batchTxID}`)
}
```

```go Go
vtxos, err := wallet.GetVtxos()
if err != nil {
    log.Fatal(err)
}

for _, vtxo := range vtxos {
    fmt.Printf("%s:%d\n", vtxo.Txid, vtxo.Vout)
    fmt.Printf("  Amount: %d sats\n", vtxo.Value)
    fmt.Printf("  State: %s\n", vtxo.VirtualStatus.State)
}
```
</CodeGroup>

### VTXO States

| State | Meaning |
|-------|---------|
| `preconfirmed` | Cosigned by operator, spendable instantly |
| `settled` | Included in onchain batch, full Bitcoin security |
| `swept` | Expired and claimed by operator (recoverable) |
| `spent` | Already used as input to another transaction |

## List Boarding UTXOs

Check pending onchain deposits that haven't been converted to VTXOs yet:

```typescript
const boardingUtxos = await wallet.getBoardingUtxos()

for (const utxo of boardingUtxos) {
  console.log(`${utxo.txid}:${utxo.vout}`)
  console.log(`  Amount: ${utxo.value} sats`)
  console.log(`  Confirmed: ${utxo.status.confirmed}`)
}
```

Once confirmed, use [Ramps](/wallets/v0.3/ramps#boarding) to convert these to spendable VTXOs.
