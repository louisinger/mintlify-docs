[Boltz](https://boltz.exchange) provides non-custodial atomic swaps between Lightning Network and Arkade. This enables seamless movement of funds between the two layers without trusting a third party.

---

## What Boltz Enables

| Capability | Description |
|------------|-------------|
| **Lightning → Arkade** | Receive Lightning payments into your Arkade wallet |
| **Arkade → Lightning** | Pay Lightning invoices from your Arkade balance |
| **Atomic execution** | Swaps complete fully or not at all—no partial states |
| **Non-custodial** | Boltz never controls your funds |

---

## How It Works

### Lightning to Arkade

```
┌─────────────┐    Lightning    ┌─────────────┐    Arkade TX    ┌─────────────┐
│   Sender    │ ──────────────► │    Boltz    │ ──────────────► │  Your VTXO  │
│ (Lightning) │                 │   (Swap)    │                 │  (Arkade)   │
└─────────────┘                 └─────────────┘                 └─────────────┘
```

1. You provide your Arkade address to Boltz
2. Boltz generates a Lightning invoice
3. Sender pays the Lightning invoice
4. Boltz sends corresponding sats to your Arkade address
5. You receive a VTXO with the swapped amount

### Arkade to Lightning

```
┌─────────────┐    Arkade TX    ┌─────────────┐    Lightning    ┌─────────────┐
│  Your VTXO  │ ──────────────► │    Boltz    │ ──────────────► │  Recipient  │
│  (Arkade)   │                 │   (Swap)    │                 │ (Lightning) │
└─────────────┘                 └─────────────┘                 └─────────────┘
```

1. You have a Lightning invoice to pay
2. Send Arkade payment to Boltz's swap address
3. Boltz pays the Lightning invoice
4. Atomic: if Lightning payment fails, your Arkade funds return

## SDK Integration

The Arkade SDK includes built-in Boltz support for Lightning operations:

```typescript
import { ArkClient } from '@arklabs/wallet-sdk';

const client = await ArkClient.init({ /* config */ });

// Receive via Lightning (creates invoice, swaps to Arkade)
const invoice = await client.createLightningInvoice({
  amount: 10000, // sats
  memo: 'Payment for services'
});

console.log('Pay this invoice:', invoice.bolt11);

// Wait for payment
await client.waitForPayment(invoice.id);
console.log('Received via Lightning → Arkade swap');
```

```typescript
// Pay a Lightning invoice from Arkade balance
const paymentResult = await client.payLightningInvoice({
  bolt11: 'lnbc10u1p...',
  maxFee: 100 // max fee in sats
});

console.log('Paid via Arkade → Lightning swap');
console.log('Fee paid:', paymentResult.feePaid, 'sats');
```

## Fee Structure

| Swap Direction | Fee |
|----------------|-----|
| Lightning → Arkade | ~0.1% + routing fees |
| Arkade → Lightning | ~0.1% + routing fees |

Fees are automatically calculated and shown before confirming swaps.

## Use Cases

### Receiving Payments

Accept Lightning payments even though your wallet uses Arkade:

```typescript
// Your app generates invoices for customers
const invoice = await client.createLightningInvoice({
  amount: orderTotal,
  memo: `Order #${orderId}`
});

// Customer pays with any Lightning wallet
// You receive VTXOs in Arkade
```

### Paying Invoices

Pay any Lightning invoice from your Arkade balance:

```typescript
// User wants to pay a Lightning merchant
const invoice = 'lnbc...'; // From merchant

await client.payLightningInvoice({ bolt11: invoice });
// Merchant receives Lightning payment
// Your Arkade balance decreases
```

### Cross-Layer Liquidity

Move funds between Lightning and Arkade based on needs:

```typescript
// Low Arkade balance? Receive Lightning payment
// Need Lightning liquidity? Pay to your own Lightning node
```
